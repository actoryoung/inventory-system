# /review-pr

## 描述
审查 Pull Request，检查代码质量、潜在问题和最佳实践。

## 使用场景
- 收到 PR 通知需要审查
- 合并代码前进行质量检查
- 团队代码审查流程

## 参数
- `pr_number` (可选): PR 编号，默认审查当前 PR

## 执行流程
1. 获取 PR 信息（标题、描述、变更文件）
2. 逐个文件检查代码质量
3. 识别潜在问题（安全、性能、可维护性）
4. 检查测试覆盖
5. 生成审查意见

## 输出格式
```markdown
## 审查总结
- 整体评价: [通过/需修改/拒绝]
- 主要问题: [列出关键问题]

## 详细意见
### 文件: path/to/file.ts
- [行号] 问题描述
- [建议修改]

## 测试建议
- [需要补充的测试]
```

## 注意事项
- **通用命令**：适用于所有代码审查
- 审查态度：建设性批评，提供可操作建议
- 关注重点：正确性 > 可读性 > 性能
