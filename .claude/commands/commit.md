# /commit

## 描述
创建符合规范的 Git 提交，自动分析变更并生成提交信息。

## 使用场景
- 完成功能开发后提交代码
- 修复 Bug 后提交代码
- 需要遵循团队提交规范时

## 参数
无（自动检测当前变更）

## 执行流程
1. 运行 `git status` 查看未跟踪文件
2. 运行 `git diff` 查看变更内容
3. 运行 `git log` 查看最近提交风格
4. 分析变更并生成提交信息
5. 执行 `git add` 和 `git commit`
6. 运行 `git status` 验证

## 注意事项
- **通用命令**：适用于所有 Git 项目
- 遵循 Conventional Commits 规范
