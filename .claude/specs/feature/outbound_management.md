# 出库管理功能规范

> 模块名称：出库管理 (Outbound Management)
> 版本：v1.0
> 创建日期：2026-01-04
> 状态：待实现

---

## 1. 功能概述

出库管理模块负责处理商品出库业务，包括出库单的创建、审核、查询等功能。出库操作会自动减少对应商品的库存数量。

### 核心功能
- 出库单 CRUD 操作
- 自动生成出库单号
- 出库自动减少库存
- 库存充足性校验
- 收货人信息管理
- 出库单状态管理

---

## 2. 业务规则

### 2.1 出库单号生成规则
- **格式**：`OUT` + `yyyyMMdd` + `4位序号`
- **示例**：`OUT202601040001`
- **规则**：
  - 单号前缀固定为 `OUT`
  - 日期部分为当天日期（yyyyMMdd）
  - 序号部分每天重置，从 0001 开始递增
  - 单号全局唯一

### 2.2 出库数量规则
- 出库数量必须 > 0
- 出库数量不能超过当前库存数量
- 出库数量不能超过最大限制（999999）

### 2.3 出库状态规则
- **待审核 (PENDING)**：新创建的出库单，待审核状态
- **已审核 (APPROVED)**：已审核通过的出库单，审核后自动减少库存
- **已作废 (VOID)**：已作废的出库单，不产生库存变化

### 2.4 库存处理规则
- 只有"已审核"状态的出库单才会减少库存
- 出库单审核通过前，必须验证库存充足
- 出库单审核通过后，自动减少对应商品的库存数量
- 作废的出库单不影响库存

### 2.5 收货人规则
- 收货人名称必填
- 收货人名称长度：1-100 字符
- 联系电话可选（11位手机号）

## 3. 数据模型

### 3.1 出库单表 (t_outbound)

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| id | BIGINT | PK, AUTO_INCREMENT | 主键 |
| outbound_no | VARCHAR(20) | NOT NULL, UNIQUE | 出库单号 |
| product_id | BIGINT | NOT NULL, FK | 商品ID |
| quantity | INT | NOT NULL, >0 | 出库数量 |
| receiver | VARCHAR(100) | NOT NULL | 收货人 |
| receiver_phone | VARCHAR(20) | | 收货人电话 |
| outbound_date | DATETIME | NOT NULL | 出库日期 |
| status | TINYINT | NOT NULL, DEFAULT 0 | 状态：0-待审核 1-已审核 2-已作废 |
| remark | VARCHAR(500) | | 备注 |
| created_by | VARCHAR(50) | | 创建人 |
| created_at | DATETIME | NOT NULL | 创建时间 |
| updated_at | DATETIME | | 更新时间 |
| approved_by | VARCHAR(50) | | 审核人 |
| approved_at | DATETIME | | 审核时间 |

### 3.2 索引设计
- PRIMARY KEY: `id`
- UNIQUE INDEX: `uk_outbound_no` ON `outbound_no`
- INDEX: `idx_product_id` ON `product_id`
- INDEX: `idx_outbound_date` ON `outbound_date`
- INDEX: `idx_status` ON `status`

## 4. API 接口设计

### 4.1 创建出库单
```
POST /api/outbound
```

**请求体：**
```json
{
  "productId": 1,
  "quantity": 50,
  "receiver": "客户A",
  "receiverPhone": "13800138000",
  "outboundDate": "2026-01-04T10:00:00",
  "remark": "备注信息"
}
```

**响应：**
```json
{
  "code": 200,
  "message": "出库单创建成功",
  "data": {
    "id": 1,
    "outboundNo": "OUT202601040001",
    "productId": 1,
    "quantity": 50,
    "receiver": "客户A",
    "status": 0,
    "createdAt": "2026-01-04T10:00:00"
  }
}
```

### 4.2 获取出库单详情
```
GET /api/outbound/{id}
```

### 4.3 获取出库单列表（分页）
```
GET /api/outbound?page=1&size=10&productId=&status=&startDate=&endDate=
```

**查询参数：**
- `page`: 页码（默认 1）
- `size`: 每页大小（默认 10）
- `productId`: 商品ID（可选）
- `status`: 状态（可选）
- `startDate`: 开始日期（可选）
- `endDate`: 结束日期（可选）

### 4.4 更新出库单
```
PUT /api/outbound/{id}
```

**限制：** 只有"待审核"状态的出库单可以修改

### 4.5 删除出库单
```
DELETE /api/outbound/{id}
```

**限制：** 只有"待审核"状态的出库单可以删除

### 4.6 审核出库单
```
PATCH /api/outbound/{id}/approve
```

**业务逻辑：**
1. 验证出库单状态为"待审核"
2. 验证库存充足
3. 更新状态为"已审核"
4. 减少对应商品的库存数量
5. 记录审核人和审核时间

### 4.7 作废出库单
```
PATCH /api/outbound/{id}/void
```

**限制：** 只有"待审核"状态的出库单可以作废

## 6. 异常处理

### 6.1 业务异常

| 异常场景 | 错误码 | 错误信息 |
|----------|--------|----------|
| 商品不存在 | 400 | 商品不存在或已禁用 |
| 出库数量不合法 | 400 | 出库数量必须大于0 |
| 库存不足 | 400 | 库存不足，当前库存为{quantity} |
| 收货人为空 | 400 | 收货人不能为空 |
| 出库单不存在 | 404 | 出库单不存在 |
| 出库单已审核 | 400 | 出库单已审核，无法修改 |
| 出库单已作废 | 400 | 出库单已作废，无法操作 |

### 6.2 系统异常
- 单号生成失败
- 库存更新失败
- 数据库异常
