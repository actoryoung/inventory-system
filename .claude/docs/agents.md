# Agents 文档

## 概述

`.claude/agents/` 目录包含了专门的 Agent 配置，每个 Agent 负责特定领域的任务。

## 使用方式

Claude Code 会根据任务类型自动选择合适的 Agent，你也可以显式指定。

## Agent 列表

### code-reviewer

**职责**：代码审查

**适用场景**：
- PR 审查
- 代码质量检查
- 最佳实践验证

**审查维度**：
1. 正确性（最高优先级）
2. 安全性
3. 可维护性
4. 性能
5. 测试

**输出格式**：
```markdown
## 审查报告
- 整体评价
- 关键问题
- 改进建议
- 亮点
- 测试建议
```

### test-writer

**职责**：编写测试

**适用场景**：
- 为新功能编写测试
- 补充缺失的测试
- 重构测试代码

**TDD 原则**：
- 测试先行
- 描述性命名
- 单一职责
- 独立性
- 可重复

### debugger

**职责**：诊断 Bug

**适用场景**：
- 报错信息分析
- 意外行为排查
- 性能问题诊断

**调试流程**：
1. 收集信息
2. 分析假设
3. 验证根因
4. 修复验证
5. 防止复发

### refactor-agent

**职责**：代码重构

**适用场景**：
- 代码坏味道清理
- 复杂度降低
- 设计模式应用

**重构目标**：
- 提高可读性
- 降低复杂度
- 减少重复
- 提高内聚
- 降低耦合

## 配置自定义 Agent

创建 `.claude/agents/my-agent.md`：

```markdown
# my-agent Agent

## 描述
Agent 的简短描述

## 适用场景
- 场景一
- 场景二

## 工具权限
- Read: 读取文件
- Write: 创建文件
- ...

## 工作原则
1. 原则一
2. 原则二

## 输出格式
定义输出模板
```
